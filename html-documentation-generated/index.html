Data Lake Mapping API

OpenAPI specification for the Data Lake ID Mapping system.

The API allows translating entity IDs between different data providers (Transfermarkt, Wyscout, StatsBomb, SkillCorner, Capology, Impect, etc.) for male players, female players, and teams.

Authentication

All requests require a token query parameter. Invalid or missing tokens return 401 Unauthorized.

ID limits

A maximum of 250 IDs can be submitted per request.

Response modes

By default the response is a structured object keyed by the source ID. Pass mode=simple to receive a flat array instead.

get /api/v1/mapping/players
Translate male player IDs between providers (getPlayerMappings)

Returns ID mappings for male players.

Supported providers (from / to)

| Short code | Full name | ID type | |------------|---------------|----------| | tm | transfermarkt | integer | | wy | wyscout | string | | sb | statsbomb | string | | sk | skillcorner | string | | cp | capology | string | | im | impect | string |

Note: When from=tm, IDs must be integers.

Query parameters

token (required)
Query Parameter — Authentication token. Must be provided on every request. Example: <code>testtokenaccess</code>
from (required)
Query Parameter — Source provider whose IDs you are translating <strong>from</strong>. Accepts both short codes and full names (case-insensitive).
to (required)
Query Parameter — Comma-separated list of target providers to translate IDs <strong>to</strong>. Example: <code>wy,sb</code> or <code>wyscout,statsbomb</code>.
ids (required)
Query Parameter — Comma-separated list of IDs in the source provider format. Maximum 250 IDs per request. Example: <code>12345,67890,11111</code>
mode (optional)
Query Parameter — Response format mode. Omit for the default structured response. Pass <code>simple</code> to receive a flat array of matched records.

Return type
